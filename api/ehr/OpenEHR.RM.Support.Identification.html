openEHR .NET

Namespace OpenEHR.RM.Support.Identification

Classes

AccessGroupRef

Reference to an access group. The namespace is "local" or a globally known access group namespace. Corresponds to ACCESS_GROUP_REF in openEHR RM 1.1.0 support.identification.

ArchetypeId

Identifier for archetypes. Instances of this class are immutable. Value format: <rm_originator>-<rm_name>-<rm_entity>.<domain_concept>.v<version_id> e.g. "openEHR-EHR-OBSERVATION.blood_pressure.v1" Corresponds to ARCHETYPE_ID in openEHR RM 1.1.0 support.identification.

GenericId

Generic identifier type for identifiers whose format is not known to openEHR. Corresponds to GENERIC_ID in openEHR RM 1.1.0 support.identification.

HierObjectId

Concrete type corresponding to the identifier of an object in a hierarchy. Used for EHR identifiers, system identifiers, etc. Corresponds to HIER_OBJECT_ID in openEHR RM 1.1.0 support.identification.

InternetId

Model of a reverse Internet domain, used as a namespace identifier. Value is a dot-separated reverse domain name, e.g. "org.openehr". Corresponds to INTERNET_ID in openEHR RM 1.1.0 support.identification.

IsoOid

Model of ISO's Object Identifier (OID) scheme of identifiers. Value is a string of dotted integers, e.g. "1.2.840.113549". Corresponds to ISO_OID in openEHR RM 1.1.0 support.identification.

LocatableRef

Reference to a LOCATABLE instance inside the top-level content structure inside a VERSION<T>, with optional path to a specific item. Corresponds to LOCATABLE_REF in openEHR RM 1.1.0 support.identification.

ObjectId

Abstract ancestor of all identifier types. Corresponds to OBJECT_ID in openEHR RM 1.1.0 support.identification.

ObjectRef

Class describing a reference to another object, using a namespace, type, and identifier. Corresponds to OBJECT_REF in openEHR RM 1.1.0 support.identification.

ObjectVersionId

Globally unique identifier for one version of a versioned object. Value format: <object_id>::<creating_system_id>::<version_tree_id> e.g. "F7C3A031-CE71-4F37-8F89::example.org::1". Corresponds to OBJECT_VERSION_ID in openEHR RM 1.1.0 support.identification.

PartyRef

Identifier for parties in a party namespace, typically from a demographic or directory service. Constrains the allowed namespaces and types. Corresponds to PARTY_REF in openEHR RM 1.1.0 support.identification.

TemplateId

Identifier for templates. A template is a further constraint on an archetype. Corresponds to TEMPLATE_ID in openEHR RM 1.1.0 support.identification.

Uid

Abstract parent of classes whose instances are unique identifiers of information entities. Corresponds to UID in openEHR RM 1.1.0 support.identification.

UidBasedId

Abstract ancestor of identifiers based on ISO/IEC 11578:1996. Value has the form: <root>[:<extension>] where root is a UID. Corresponds to UID_BASED_ID in openEHR RM 1.1.0 support.identification.

Uuid

Model of the DCE Universal Unique Identifier (UUID) scheme of identifiers. Value is a string of the form: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx Corresponds to UUID in openEHR RM 1.1.0 support.identification.

VersionTreeId

Identifier of a version in a version tree. Value format: <trunk_version>[.<branch_number>.<branch_version>] e.g. "1", "1.1.1", "2.1.3". Corresponds to VERSION_TREE_ID in openEHR RM 1.1.0 support.identification.
